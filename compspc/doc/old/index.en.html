COMPSPC

COMPSPC is a simple program to analyze and convert music. Yes, this is a MIDI converter.

Simple Usage for Beginners

  1. Put compspc.exe and allconv.bat in target directory
  2. Run allconv.bat, then you will get MIDI files soon
  3. Have fun! ;)

This usage is for Windows. I don't know other any platforms. :P

Usage

COMPSPC is a console application. I show the syntax to the following.

compspc <spc files>

MIDI files named from ID666 will be created, and translation logs will be put to standard output.

Notes

  • SNESmusic.org SPC sets are recommended for some reasons
  • Neither jingles nor sound effects will be converted perhaps
  • Has some conversion issues
  • For research, not for playback
  • Source codes included, in the public domain

Why SNESmusic.org SPC sets?

Because my favorite, and the following reason.

  • ID666 timers and tags are set
  • Each set uses same hack code

COMPSPC obtains the version of the music engine from ID666 automatically, and uses ID666 timers to stop conversion (if no timers, converts 180 seconds). Moreover, COMPSPC is for hacked SPCs because of NTSC/PAL switching (read timer0 latch).

Why supports music only?

COMPSPC searches music table, and converts music 01 always. Usually, music that SPC player plays is 01, but neither jingles nor sound effects are so. I don't know how to get correct song. I know the way to get playing position, but I don't want to use this method (because some events might not be dumped). Fix it if you can.

Other conversion issues

  • Difference of tempo
  • Tuning problem

The music engine has independent tempo value of each track, but MIDI file is not so. This difference might cause the problem around playback (I have not seen such cases however).

The music engine has tuning event, and tuning value is added to DSP pitch registers. The RPN tuning cannot emulate this. Converter can emulate it by calculating the pitch at each note, but I don't want it.
